## 架构偏好与分层原则

- UI 与数据分离。
- UI 层只负责展示、用户交互入口和状态渲染，不直接承载数据访问和业务规则。
- 状态协调层，例如 ViewModel、Presenter、Controller 或等价模式，负责把领域状态转换成 UI 状态。
- 数据访问、持久化、网络请求和外部服务封装在数据层或服务层。
- 小范围 UI 调整不应穿透修改数据层；如果必须穿透，先记录原因和影响范围。
