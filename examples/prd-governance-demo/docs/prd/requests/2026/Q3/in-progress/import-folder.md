## 背景

老项目已经支持首次扫描目录，但用户后续往目录里新增照片后，需要能重新扫描并只处理新增或变化文件。

## 目标

- 用户可以对已接入目录重新扫描。
- 系统只处理新增、修改或删除的文件。
- 扫描结果要展示新增、更新、移除和失败数量。

## 非目标

- 不做实时文件监听。
- 不做云同步。
- 不做图片内容分析。

## 用户流程

1. 用户进入目录详情页。
2. 点击重新扫描。
3. 系统显示扫描进度。
4. 扫描结束后展示变化摘要。

## 验收标准

- 重复扫描同一个未变化目录，不应重复创建索引记录。
- 新增图片会出现在照片列表。
- 删除的文件会被标记为 missing，而不是直接清除历史记录。
- 失败文件会进入错误列表。
