# 03-技术方案

plan_status: review
migration_level: compatible

## 技术目标

实现导入目录的增量扫描：基于目录快照比较 path、mtime、size，输出新增、更新、缺失和失败集合。

## 当前代码风格

导入逻辑放在 `src/import/**`，索引写入放在 `src/index/**`，测试放在 `tests/import/**`。扫描和写入分层，避免扫描函数直接写索引。

## 架构偏好与分层约束

- UI 层只触发扫描动作并展示扫描状态，不直接读取目录快照。
- 状态协调层负责把扫描结果转换成 `idle`、`scanning`、`writing`、`completed`、`failed`。
- 数据 / 服务层负责目录扫描、快照比较和索引写入。
- 小 UI 改动只能调整状态展示和交互入口，不应修改扫描算法或索引结构。

## 影响范围

- `src/import/**`
- `src/index/**`
- `tests/import/**`

## 非影响范围

- 不修改 `package.json`。
- 不引入文件监听。
- 不增加云同步。

## 数据结构

维护目录快照：

- `path`
- `mtime`
- `size`
- `status`: `new` / `updated` / `missing` / `unchanged`

## API / 函数边界

- `src/import/scanDirectory` 只负责生成变化集合。
- `src/index/writeImportResult` 只负责写入索引。

## 状态流

`idle -> scanning -> writing -> completed`。扫描失败进入 `failed`，允许用户重试。

## 错误处理

- 单文件读取失败记录到失败集合，不中断整个扫描。
- 目录不可访问时终止本次扫描并返回错误。

## 测试策略

- 目录无变化重复扫描。
- 新增文件扫描。
- 删除文件标记 missing。
- 单文件读取失败。

## 兼容性和迁移

兼容现有索引结构，只新增缺失状态字段，不需要破坏性迁移。

## 风险和回滚

- 风险：mtime 精度差异可能导致误判。
- 回滚：回滚增量扫描分支，保留旧的全量扫描路径。
